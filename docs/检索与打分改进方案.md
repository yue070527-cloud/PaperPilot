## 二、改进方案详情

### P0：Cross-Encoder 重排序

**目标**：解决 FAISS 打分区分度不足的问题

**原理**：
- bi-encoder：分别编码查询和文档，通过向量内积计算相似度——快但粗糙
- cross-encoder：将查询和文档拼接后联合编码——慢但精确
- 标准 IR 范式：bi-encoder 召回 → cross-encoder 精排

**技术方案**：
```
论文抓取 → FAISS bi-encoder 粗筛 top-K → cross-encoder 精排 → 终榜
```

**推荐模型**：`cross-encoder/ms-marco-MiniLM-L-6-v2`（~80MB，CPU 友好）

**输入格式**：
```python
# (课题描述, 论文摘要) 配对
pairs = [(query_en, paper["abstract"]) for paper in candidates]
scores = cross_encoder.predict(pairs)
```

**性能估算**：50 篇论文 × 384 token ≈ 3-5 秒（CPU）

**归属**：A — `paperpilot/indexer.py`

**改动量**：~80 行新函数 `rerank_with_cross_encoder(query, papers, top_k)`

---

### P1：AND/OR 分层查询

**目标**：核心关键词必须命中，普通关键词提升召回

**现状**：
```python
# 当前：全部 OR
_build_search_query(["高熵电解质", "水系锌离子电池", "设计策略"])
→ "高熵电解质" OR "水系锌离子电池" OR "设计策略"
```

**目标格式**：
```
"核心词1" AND "核心词2" AND ("普通词1" OR "普通词2" OR ...)
```

**数据基础**（已有，无需改动）：
| 层级 | 来源 | 权重 | 数量 | 用途 |
|------|------|------|------|------|
| 核心关键词 | `extract_core_keywords` | 1.0 | 1-3 | AND 连接 |
| 普通关键词 | `extract_regular_keywords` | 0.75 | 5-8 | OR 连接 |

**AI 角色界定**：
- AI 负责区分"核心/普通"两级（已完成，有权重数据）
- AI 不负责"主/副"细分——避免错误预判
- 若需更精细控制，用户在 UI 手动指定主关键词

**归属**：
- A — `paperpilot/fetcher.py` `_build_search_query` 支持分层参数
- B — `paperpilot/app.py` `_run_pipeline` 传递分层关键词列表

**改动量**：A ~20 行 + B ~10 行

### P3：API 原始分数保留与融合

**目标**：利用 arXiv/OpenAlex 自带的全文相关性排序

**现状证据**：
```python
# arXiv: 已按相关性排序，但分数未保存
arxiv.Search(query=query, sort_by=arxiv.SortCriterion.Relevance)

# OpenAlex: 默认 relevance 排序，分数同样丢失
params = {"search": query, "per_page": per_page}
```

**方案**：

1. **fetcher 层**：在 paper dict 中新增 `api_score` 字段
   - arXiv：归一化后的排序位置（`1 - idx/max_results`）
   - OpenAlex：API 返回的 `relevance_score`（若有）或排序位置

2. **indexer 层**：融合分数
   ```python
   final_score = α × api_relevance + β × cross_encoder_score
   # α ≈ 0.3, β ≈ 0.7（可调）
   ```

**归属**：
- A — `fetcher.py` 加 `api_score` 字段
- A — `indexer.py` 实现分数融合

**改动量**：~30 行

---

### P4：OpenAlex 精细化过滤

**目标**：利用 OpenAlex 的 filter 参数缩小检索范围

**可用 filter 参数**：
| 参数 | 说明 | 示例 |
|------|------|------|
| `filter` | 年份范围 | `publication_year:2020-2025` |
| `filter` | 引用量阈值 | `cited_by_count:>10` |
| `filter` | 学科领域 | `concepts.id:C123456` |
| `filter` | OA 状态 | `is_oa:true` |
| `filter` | 文献类型 | `type:journal-article` |

**归属**：A — `fetcher.py` + B — UI 设置页

**改动量**：~40 行

## 四、风险与边界

- **Cross-encoder 不能变出论文**：如果抓回来的论文确实不相关，cross-encoder 只能给低分，依赖上游检索质量
- **AND 查询可能太严**：核心关键词只有 1-3 个，都是 AND 连接可能导致 0 结果，需要降级策略（0 结果时回退 OR）
- **分数融合的 α/β 权重需要调优**：建议先在 10-20 个课题上人工评测，确定最佳比例
- **cross-encoder 模型首次加载需下载**：约 80MB，首次搜索会慢 10-20 秒
