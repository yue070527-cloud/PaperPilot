## 1. 项目简介

PaperPilot 是一款**桌面端学术文献管理工具**，帮助你完成从"课题输入"到"文献阅读"的完整工作流：

- 输入研究课题 → 自动提取关键词 → 从 arXiv 和 OpenAlex 检索论文 → 智能排序打分 → 保存到文献库 → AI 精读分析 → 导出 BibTeX/CSV

**核心亮点：**

- **智能关键词提取**：AI 自动从课题描述中提取核心技术术语，支持手动拖拽归类（主/副/普通三层）
- **双源检索**：同时搜索 arXiv（预印本）和 OpenAlex（正式发表论文），自动去重
- **语义精排**：使用 Cross-Encoder 模型对检索结果进行语义相关性排序
- **AI 精读**：调用 DeepSeek API 对论文全文进行结构化分析（核心贡献 / 方法 / 证据 / 亮点 / 局限）
- **PDF 阅读器**：内置 PDF.js 阅读器，支持文字选择、搜索、页码导航、夜间模式
- **AI 对话助手**：课题上下文感知的学术对话助手，支持 Markdown 富文本渲染

## 3. 安装步骤

以下步骤假设你使用的是 Windows 系统。下面出现的命令请在 **命令提示符（cmd）**或 **PowerShell** 中执行。

### 3.1 下载项目代码

**方式一：Git 克隆（推荐）**

打开命令行，进入你想存放项目的目录（例如 `E:\`），执行：

```bash
git clone https://github.com/yue070527-cloud/PaperPilot.git
cd PaperPilot
```

**方式二：直接下载 ZIP**

1. 在浏览器打开 https://github.com/yue070527-cloud/PaperPilot
2. 点击绿色的 **Code** 按钮 → **Download ZIP**
3. 解压到你想要的目录（例如 `E:\PaperPilot`）

### 3.2 创建虚拟环境（推荐但可选）

虚拟环境可以隔离这个项目的依赖，不让它和系统其他 Python 项目冲突。

```bash
python -m venv .venv
.venv\Scripts\activate
```

激活成功后，命令行前面会出现 `(.venv)` 标识。

### 3.3 安装依赖

```bash
pip install -r requirements.txt
```

这会安装以下主要依赖：

| 包名 | 用途 |
|------|------|
| flet | 桌面 UI 框架 |
| sqlalchemy | 数据库 |
| faiss-cpu | 向量检索 |
| sentence-transformers | 语义模型 |
| keybert | 英文关键词提取 |
| arxiv | arXiv API 客户端 |
| pymupdf | PDF 文本提取 |
| pywebview | PDF 独立阅读窗口 |
| jieba | 中文分词 |
| beautifulsoup4 + lxml | HTML 解析 |
| pyyaml | 配置文件解析 |

> **安装时间**：首次安装约需 3–8 分钟（含 PyTorch 自动下载）。如果下载慢，可以配置 pip 镜像：
> ```bash
> pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
> ```

> **关于 PDF 自动下载**：程序会优先使用 `curl_cffi`（更强的浏览器指纹伪装）访问出版商网站获取 PDF。如果 `curl_cffi` 未安装，程序会自动降级为普通 HTTP 请求，不影响基本使用。如需安装：
> ```bash
> pip install curl_cffi
> ```

### 3.4 创建配置文件

项目目录下有一个配置模板文件 `config.example.yaml`，你需要复制它并填入你自己的配置。

```bash
copy config.example.yaml config.yaml
```

然后用记事本（或其他文本编辑器）打开 `config.yaml`，修改以下内容：

```yaml
deepseek:
  api_key: sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx  # 替换为你的 DeepSeek API Key
  model: deepseek-v4-flash                       # 模型选择，默认即可

data_sources:
  arxiv: true      # 是否从 arXiv 检索（需要网络）
  openalex: true   # 是否从 OpenAlex 检索（需要网络，但免费免 Key）

ui:
  theme: cyan      # 默认主题色（可选 mint/ocean/sand/dusk/rose/cyan）
  dark_mode: true  # 是否默认夜间模式
```

> **如何获取 DeepSeek API Key？**
> 1. 打开 https://platform.deepseek.com/
> 2. 注册并登录
> 3. 进入 **API Keys** 页面 → 点击 **创建 API Key**
> 4. 复制生成的 Key（以 `sk-` 开头），粘贴到 `config.yaml` 的 `api_key` 位置
> 5. DeepSeek 新用户通常有免费额度，充值 10 元左右即可长期使用

如果你暂时没有 API Key，把 `api_key` 留空（不建议！）也可以使用基本功能（关键词提取会回退到本地算法，但 AI 精读和对话助手不可用）。

## 4. 首次启动

确保你已完成上述所有步骤后，在项目目录下执行：

```bash
python app.py
```

### 首次启动会发生什么

1. **窗口出现**：一个约 1200×750 的桌面窗口弹出，默认打开"文献"页面
2. **数据库创建**：项目目录下自动生成 `paperpilot.db`（SQLite 数据库文件）
3. **模型下载（首次检索时）**：

   当你第一次点击"开始检索"时，程序会自动下载以下模型文件：

   | 模型 | 大小 | 用途 | 存放位置 |
   |------|------|------|---------|
   | mxbai-rerank-base-v2 | ~942 MB | 语义精排 | `~/.cache/modelscope/` 或 HuggingFace 缓存 |
   | paraphrase-multilingual-MiniLM-L12-v2 | ~470 MB | 中文关键词提取（回退用） | 同上 |
   | all-MiniLM-L6-v2 | ~90 MB | 英文关键词提取（回退用） | 同上 |

   > 这三个模型仅在 **DeepSeek API 不可用** 或 **进行 CE 语义排序** 时才会下载。如果你已配置 API Key 且不点"CE 排序"按钮，可能永远不需要下载它们。
   >
   > 下载时间取决于网络（校园网/国内网络较慢，首次可能需要 10-30 分钟）。下载只需一次，后续使用不再重复下载。

4. **PDF.js 下载（首次阅读 PDF 时）**：

   第一次点击"阅读原文"时，程序会自动下载 PDF.js 渲染引擎（~3 MB），缓存到 `~/.paperpilot_pdfjs/` 目录。之后即使断网也能正常使用 PDF 阅读器。

### 启动成功标志

看到以下界面即表示启动成功：

- 顶部导航栏有"文献 / 检索 / 设置"三个标签
- 右侧有"StudyCopilot"对话面板

## 5. 功能详解

### 5.1 检索论文

这是核心工作流的第一步，帮助你从海量论文中找到与课题相关的文献。

**操作步骤：**

1. 点击顶部导航栏的 **"检索"** 标签
2. 在 **课题名称** 输入框填写研究课题的简短名称（例如"钙钛矿太阳能电池稳定性"）
3. 在 **课题描述** 输入框填写 1–3 句话描述你的研究方向（越具体越好）
4. 点击 **"提取关键词"** 按钮——AI 会自动分析你的课题描述，提取出技术关键词
5. 提取完成后，关键词会出现在三个拖拽区中：
   - **主关键词**（星标）：检索的核心词，必须命中
   - **副关键词**（箭头）：辅助检索词
   - **普通关键词**：补充召回
6. 你可以**拖拽关键词 Chip** 在三个区域之间移动，调整它们的重要性。也可以手动添加新的关键词
7. 点击 **"开始检索"** 按钮

**检索过程：**

- 程序先将中文关键词翻译为英文
- 然后分别从 arXiv 和 OpenAlex 检索
- 各来源结果去重合并
- 最后用 Cross-Encoder 语义模型精排

**检索结果：**

默认每个数据源检索 **250 篇**（可在设置中调整）。结果列表显示每篇论文的标题、作者、年份、来源、引用次数、CE 得分和文章类型（综述/研究论文）。你可以：

- 点击任意一篇论文查看详情（右侧侧边栏弹出）
- 点击表头（标题/作者/年份/引用/得分）进行排序
- 使用 **"多选"** 模式勾选感兴趣的论文
- 点击 **"保存到文献库"** 将结果存入某个课题

**AI 精排（可选）：**

检索完成后，你可以点击 **"AI 精排"** 按钮，让 AI 基于摘要对结果进行四维打分（相关性 × 方法质量 × 创新性 × 时效性），输出 0-100 分的综合评分和 S/A/B/C/D 档位，帮助你快速锁定最值得读的论文。

> AI 精排需要 DeepSeek API Key，按 token 计费，每篇论文约消耗 ~120 tokens。

### 5.2 管理文献库

点击顶部 **"文献"** 标签进入文献管理页面。

**课题管理：**

- 左侧显示所有课题列表，点击可切换
- 点击 **"+"** 按钮新建课题
- 点击课题旁的菜单进行重命名或删除
- 删除课题时 PDF 文件会进入回收站（7 天后自动清理）

**论文管理：**

- 论文列表显示标题、作者、年份、AI 得分、CE 得分、阅读状态
- 状态筛选：全部 / 未读 / 略读 / 精读
- 点击状态指示灯切换阅读状态（未读→略读→精读→未读）
- 点击论文标题查看完整详情（弹出对话框，含摘要、AI 评分各维度理由、AI 精读笔记等）
- 已下载 PDF 的论文序号和标题会显示为**绿色**，方便识别
- 点击 **阅读按钮**（📖）打开 PDF 阅读器
- 点击 **AI 精读按钮**（🧠）启动 AI 结构化分析
- 点击 **删除按钮**（🗑）移除论文

**导入本地 PDF：**

- 点击 **上传按钮**（📤），可以选择：
  - 选择单个 PDF 文件
  - 选择多个 PDF 文件
  - 选择整个文件夹（自动递归扫描所有 PDF）
- 导入时程序会用 PyMuPDF 自动提取标题（支持按 PDF 内最大字号定位）、作者、摘要和年份
- 扫描件（纯图片无文字层的 PDF）会被自动跳过
- 编码乱码的 PDF metadata 标题会被自动过滤并回退到字号检测

**排序：**

- 点击 **排序按钮**（⇅）用 Cross-Encoder 对课题内论文重新进行语义排序
- 点击 **AI 排序按钮**（📊）用 AI 基于摘要对论文打分排序
- 两种排序结果都会持久化保存到数据库

**多选与对比：**

- 点击 **"多选"** 进入多选模式，勾选论文后可批量删除或**对比分析**（选中 2 篇以上后点击对比按钮，Agent 会从研究目标、方法、发现、创新点、局限性五个维度进行对比）
- 在文献库多选论文后，Agent 对话面板也会自动将选中论文作为上下文

**保存机制：**

- 检索结果保存到文献库时，仅保存**当前显示的前 N 篇**（即排序后的 top 结果），而非全部检索到的论文
- 保存后程序会**后台自动下载**可获取的 PDF，无需手动操作

### 5.3 AI 精读与对话

**AI 精读一篇论文：**

在文献库中点击论文旁的 **🧠 按钮**，AI 会：

1. 自动获取论文全文（优先 PDF → 网页 HTML → 摘要回退）
2. 用 RLM 三层阅读策略分析：
   - 短文本（<8K 字符）：直接全文分析
   - 中等文本（8K–60K）：滑动窗口分段阅读
   - 长文本（>60K）：切块独立分析后合成
3. 输出结构化分析报告：
   - 核心贡献：论文解决的关键问题
   - 研究方法：技术路线
   - 关键证据：支撑结论的实验/数据
   - 创新亮点
   - 局限不足
   - 评分：新颖性 / 严谨性 / 重要性（各 1-10 分）

分析结果会展示在右侧 StudyCopilot 面板中，同时保存到数据库和本地 `outputs/deep_read/` 目录。

**AI 对话助手（StudyCopilot）：**

右侧常驻的 **StudyCopilot** 面板是一个课题上下文感知的学术对话助手，支持 **Markdown 富文本渲染**（标题、粗体、列表、表格、代码块）。

- 选择一个课题后，助手会自动加载该课题的论文信息
- 你可以直接问："这篇论文的方法有什么亮点？" / "对比一下前两篇论文" / "这个研究方向还有哪些值得关注的子方向"
- 如果你在消息中 **@论文标题** 或包含论文标题关键词，助手会自动引用该论文的详细信息
- **Agent 可主动执行操作**：在对话中可以要求 Agent "把这个课题改名为xxx" / "把课题描述改成xxx" / "开始新的检索"，Agent 会自动弹出确认对话框并执行
- **面板可拖拽调整宽度**：拖动 Agent 面板左边缘可调整宽度
- 对话历史会自动保存，即使关闭程序也不会丢失
- 当对话过长（超过约 80K token）时，旧消息会自动压缩为摘要，不影响后续对话质量

### 5.4 阅读原文

点击论文旁的 **📖 按钮**，程序会按以下优先级尝试获取全文：

1. **本地 PDF 文件**：如果你已导入 PDF，直接打开
2. **课题仓库**：查找该论文是否已在其他课题中导入过
3. **下载缓存**：查找之前是否通过检索页下载过
4. **自动下载**：对于 arXiv、Nature、Springer 等期刊，尝试直链下载 PDF（优先使用 `curl_cffi` 浏览器指纹伪装，未安装时自动降级为普通请求）
5. **无法获取**：弹出对话框，引导你手动下载

打开后，PDF 会在一个**独立窗口**中渲染：

- 支持鼠标滚轮滚动翻页
- 支持文字选择（拖动选中，Ctrl+C 复制）
- 顶部导航栏显示当前页码，可跳转到指定页
- 支持上一页 / 下一页按钮
- 主题颜色跟随主界面设置（夜间模式自动适配深色背景）

> PDF 阅读器基于 PDF.js 3.11，首次使用时自动下载 ~3MB 文件到本地，之后无需网络。阅读器使用 Edge WebView2 渲染（Windows 10/11 自带）。

### 5.5 导出文献

在文献管理页面，点击工具栏的 **下载按钮**（📥），可以选择：

| 格式 | 说明 |
|------|------|
| **BibTeX** | LaTeX 文献引用格式，可直接用于 Overleaf / 本地 LaTeX 项目 |
| **CSV** | Excel 表格格式，含标题、作者、年份、来源、期刊、引用数、DOI、URL |

导出时会弹出保存对话框，让你选择保存位置和文件名。如果是多选模式下导出，只导出选中的论文。

### 5.6 设置页面

点击顶部 **"设置"** 标签进入设置页面。

| 设置项 | 说明 |
|--------|------|
| **DeepSeek API Key** | 用于关键词提取、AI 精读、对话助手。以 `sk-` 开头 |
| **模型选择** | DeepSeek V4 Flash（推荐）/ V3（7月将下线） |
| **数据源** | 开关 arXiv / OpenAlex 检索 |
| **检索数量** | 每个来源的最大检索结果数（100–500 篇，默认 250） |
| **显示/精排数量** | 最终显示论文数（10–200）和送入精排的候选数（10–200） |
| **配色主题** | 6 种主题色：薄荷绿 / 海蓝 / 暖沙 / 暮紫 / 玫瑰 / 青碧 |
| **夜间模式** | 切换深色/浅色界面 |

所有设置自动保存到 `config.yaml`，下次启动自动恢复。

## 6. 常见问题

### Q1：启动时报错 "No module named 'xxx'"

依赖没有完整安装。执行：

```bash
pip install -r requirements.txt
```

如果某个包安装失败，尝试单独安装它：

```bash
pip install 包名
```

### Q2：点击"提取关键词"后卡住不动

可能是 DeepSeek API Key 未配置或网络不通。

- 检查 `config.yaml` 中的 `api_key` 是否正确
- 如果确实没有 API Key，把 `api_key` 留空，程序会回退到本地算法（需要下载模型，首次较慢）

### Q3：检索论文时提示"未找到相关论文"

- 检查网络连接（arXiv 和 OpenAlex 都需要网络）
- 尝试在设置中减少"精排候选"数量
- 尝试在设置中增加每个数据源最大检索数量（检索数量建议选 250 篇或以上，不建议小于 100 篇）
- 检查关键词翻译是否正确（中文关键词需要有对应的英文翻译）
- 尝试只开启一个数据源（先只用 openalex 试试）

### Q4：Cross-Encoder 模型下载太久/失败

这通常是因为网络到 HuggingFace 不通畅。可以尝试以下方法：

- **方法一**：设置 HuggingFace 镜像（国内用户推荐）
  在命令行执行：
  ```bash
  set HF_ENDPOINT=https://hf-mirror.com
  ```
  然后重启程序。

- **方法二**：手动下载模型
  1. 从 https://hf-mirror.com/mixedbread-ai/mxbai-rerank-base-v2 下载所有文件
  2. 放入 `C:\Users\你的用户名\.cache\modelscope\mixedbread-ai\mxbai-rerank-base-v2\` 目录

- **方法三**：不用 CE 排序
  程序在 CE 超时后会**自动回退到 API 分数排序**，不影响使用。你可以在设置中将"精排候选"调低。

### Q5：PDF 阅读器打不开/闪退

- 阅读器使用 Edge WebView2 运行时（Windows 11 自带，Windows 10 可能需要手动安装）：
  下载地址：https://developer.microsoft.com/en-us/microsoft-edge/webview2/
- 确认已安装 `pywebview`：`pip install pywebview>=4.0`

### Q6：Wiley / ACS / IEEE 等期刊的 PDF 下载不了

这些期刊有付费墙，直链下载只对**你所在机构已订阅**的期刊有效。arXiv 论文通常可以免费下载。

- 如果安装了 `curl_cffi`（`pip install curl_cffi`），程序会使用更强的浏览器指纹伪装，有助于部分出版商的 PDF 下载
- 解决方法：在浏览器中登录学校/机构的 VPN 或图书馆代理，手动下载 PDF，然后通过文献库的"导入 PDF"功能添加到 PaperPilot

### Q7：程序占了 2GB+ 内存

这是正常的，主要内存占用来自：

- Cross-Encoder 模型（~942 MB），在检索和 CE 排序时会加载
- PyTorch 运行时（~500 MB）
- PDF 阅读器窗口（~200 MB）

如果内存紧张，检索完成后程序会自动释放 Cross-Encoder 模型。你可以在设置中将"精排候选"数调低以减少内存峰值。

### Q8：如何更新到最新版本？

如果你使用 Git 克隆的：

```bash
git pull origin main
pip install -r requirements.txt  # 更新依赖（如有变化）
```

如果是 ZIP 下载的：重新下载最新 ZIP 并覆盖，注意保留你的 `config.yaml`（不要覆盖它）。

### Q9：数据库文件在哪里？可以备份吗？

数据库文件在项目根目录下的 `paperpilot.db`。直接复制这个文件即可备份。

论文 PDF 存储在项目根目录下的 `repository/` 文件夹中，按课题名称分目录存放。

### Q10：如何完全卸载？

删除项目文件夹即可。此外以下目录也可能存有缓存文件，可手动删除：

- `C:\Users\你的用户名\.paperpilot_pdf_cache\`（PDF 下载缓存）
- `C:\Users\你的用户名\.paperpilot_pdfjs\`（PDF.js 引擎）
- `C:\Users\你的用户名\.paperpilot_html_cache\`（HTML 全文缓存）
- `C:\Users\你的用户名\.cache\modelscope\`（AI 模型缓存，与其他项目共享）
